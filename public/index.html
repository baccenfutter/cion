This is XCion.Cloud - a DynDNS SaaS provider for the internet of things.

XCion.Cloud offers a free DynDNS service for A, MX, SRV, TXT and CNAME records with no strings attached. Users can easily register namespaces and then create and manage records by simply sending HTTP requests.

XCion.Cloud leverages the popular open-source nameserver Bind9 and really only translates the HTTP requests into dynamic DNS updates (RFC-2136) which it then executes locally.

All records reside within a single Bind9 zonefile which is replicated to a secondary nameserver whenever an update is performed on the zone.



Usage

Registering a Namespace

Registering namespaces follows the first-come-first-serve principle. Simply send a PUT request to /register, stating the desired namespace as parameter. 

curl \
  -X PUT \
  -H "Accept: application/json; version=1.0.0" \
  -H "Content-Type: application/json" \
  -d '{"zone": "example"}' \
  https://xcion.cloud/register

If the desired namespace has already been registered by another user, the call will return an HTTP-423 (LOCKED). In that case, simply try registering for a different namespace instead. If the namespace is still vacant it returns an HTTP-202 (ACCEPTED) together with an authentication token that will be needed for managing your zone. Pass it as X-Cion-Auth-Key in all of your subsequent requests to authenticate as owner for the zone.

Please do keep a secure backup of this token as loosing it will result in loss of control over your registered namespace!

Namespaces that do not receive any zone updates for 60 days, will automatically be garbage-collected. After another week the namespace will be deleted and made re-available for registration.

A-type records

To create an A-type record within your zone, send a POST requests as follows: 

curl \
  -X POST \
  -H "Accept: application/json; version=1.0.0" \
  -H "Content-Type: application/json" \
  -H "X-Cion-Auth-Key: ..." \
  -H "X-Cion-Update-Type: A" \
  -d '{"hostname":"www","address":"127.0.0.1"}' \
  https://xcion.cloud/zone/example

If a record with that hostname already exists, it is overwritten with the new address. Multiple A-type records with the same hostname are not supported.

MX-type records

To create an A-type record within your zone, send a POST request as follows: 

curl \
  -X POST \
  -H "Accept: application/json; version=1.0.0" \
  -H "Content-Type: application/json" \
  -H "X-Cion-Auth-Key: ..." \
  -H "X-Cion-Update-Type: MX" \
  -d '{"pref":"10","name:"mx1.example.com"}' \
  https://xcion.cloud/zone/example

Passing a hostname parameter for MX-type records is not supported. They are always created directly on you *.xcion.cloud zone. If an MX-type record with the same preference (pref) already exists, it is overwritten with the new name. Multiple MX-type records are only supported if they have differing preferences.

Please also note that the request will not be accepted if no A-type record could be found for the name specified in the destination parameter.

SRV-type records

To create an SRV-type record within your zone, send a POST request as follows: 

curl \
  -X POST \
  -H "Accept: application/json; version=1.0.0" \
  -H "Content-Type: application/json" \
  -H "X-Cion-Auth-Key: ..." \
  -H "X-Cion-Update-Type: SRV" \
  -d '{"srv":"matrix","proto":"tcp","prio":"10","weight":"0","proto":"8448","dest"}' \
  https://xcion.cloud/zone/example

If another SRV-type record with the same srv, proto, prio and weight already exists, it is overwritten with the new port and dest. Multiple SRV-type records with the same srv and proto are only supported with differing priorities and weights.

TXT-type records

To create TXT-type records within your zone, send a POST requst as follows: 

curl \
  -X POST \
  -H "Accept: application/json; version=1.0.0" \
  -H "Content-Type: application/json" \
  -H "X-Cion-Auth-Key: ..." \
  -H "X-Cion-Update-Type: SRV" \
  -d '{"gpg-id":"","value":"0x000000"}' \
  https://xcion.cloud/zone/example

TXT-type records do not support a hostname parameter, instead all records are always create directly within your *.xcion.cloud namespace. If another TXT-type record with the same name already exists, it is overwritten with the new one.

CNAME-type records

To create a CNAME record within your zone, send a POST request as follows: 

curl \
  -X POST \
  -H "Accept: application/json; version=1.0.0" \
  -H "Content-Type: application/json" \
  -H "X-Cion-Auth-Key: ..." \
  -H "X-Cion-Update-Type: CNAME" \
  -d '{"name":"www1","dest":"www"}' \
  https://xcion.cloud/zone/example

Note, that the destination parameter will always be prepended with <your-zone>.xcion.cloud. in order to prevent abuse (i.e. for fishing). CNAME-type records pointing to destinations outside of your zone is not supported.
