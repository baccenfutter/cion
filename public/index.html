This is XCion.Cloud - a DynDNS SaaS provider for the internet of things.

XCion.Cloud provides an easy to use HTTP interface with which users can register their own public namespace at *.xcion.cloud for managing and regularly updating their dynamically changing SRV records.

XCion.Cloud leverages the popular open-source nameserver Bind9 and really only translates the HTTP requests into dynamic DNS updates (RFC-2136) which it then executes locally.



Usage

Registering a Namespace

Registering namespaces follows the first-come-first-serve principle. Simply send a PUT request to /register, stating the desired namespace as parameter. 

curl \
  -X PUT \
  -H "Accept: application/json; version=1.0.0" \
  -H "Content-Type: application/json" \
  -d '{"zone": "example"}' \
  https://xcion.cloud/register

If the desired namespace has already been registered by another user, the call will return an HTTP-423 (LOCKED). In that case, simply try registering for a different namespace instead. If the namespace is still vacant it returns an HTTP-202 (ACCEPTED) together with an authentication token that will be needed for managing your zone.

Please do keep a secure backup of this token as loosing it will result in loss of control over your registered namespace!

Namespaces that do not receive any zone updates for 60 days, will automatically be garbage-collected. After another week the namespace will be deleted and made re-available for registration.

Creating and updating records

Once the zone has been successfully registered SRV records can be added and updated by POSTing some request parameters as follows. Please remember to always pass your X-Cion-Auth-Key to authorize your requests. 

curl \
  -X POST \
  -H "Accept: application/json; version=1.0.0" \
  -H "Content-Type: application/json" \
  -H "X-Cion-Auth-Token: 0123456789abcdef" \
  -d '{"srv":"matrix","proto":"tcp","prio":10,"weight":0,"port":8448,"dest":"127.0.0.1"}' \
  https://xcion.cloud/zone/example

The request parameters are:

  • srv - Name of the service (without the preceding underscore)
  • proto - Protocol under which the service is available (again, no underscore)
  • prio - Record priority (lower values have higher priority)
  • weight - Record weight (higher values have higher priority)
  • port - Port under which your serice is available
  • dest - Endpoint of your service (IP or hostname / hostnames must end with period)

For more details on the record fields see: Wikipedia.

The above request would create the following DNS record: 

_matrix._tcp.example.xcion.cloud. 180	IN	SRV	10 0 8448 127.0.0.1
